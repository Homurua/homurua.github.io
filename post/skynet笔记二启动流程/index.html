Skynet笔记（二）启动流程

2022-02-10

    • main
    • skynet_start

　　本篇大概讲一下 skynet 进程的启动流程，完全了解启动流程基本上也就明白了 skynet 的原理了，本篇会大概提到全部的启动流程。

main

  1. 根据参数拿到配置文件的路径
  2. 初始化全局数据结构 skynet_node *G_NODE
  3. 初始化全局环境变量 skynet_env *E
  4. 忽略信号 SIGPIPE
  5. 创建一个虚拟机加载一段硬代码（hard code）进去并且加载配置文件
  6. 把读取到的配置文件的内容写入到环境变量中
  7. 从环境变量中拿到 skynet_config config 需要的值初始化它
  8. 把加载配置创建出来的虚拟机关闭
  9. 把 config 的地址作为参数传给 skynet_start 函数继续后面的步骤

skynet_start

  1. 处理信号 SIGHUP
  2. 处理 daemon 相关的配置，如果配置了 daemon 启动的话，调用 daemon_init 处理 daemon 相关的操作
  3. 初始化 harbor 节点数量
  4. 初始化 handle 管理器 handle_storage *H
  5. 初始化全局消息队列 global_queue *Q
  6. 初始化 C 模块管理器 modules *M
  7. 初始化全局时间管理器 timer *TI
  8. 初始化全局 socket 管理器 socket_server *SOCKET_SERVER
  9. 标记性能测试标志
  10. 创建 config 中的 logger 服务
  11. 调用 bootstrap函数，传入配置中的 bootstrap 字段内容
  12. 调用 start 函数，传入配置中的 thread 字段

bootstrap

  1. 根据配置中的 bootstrap 字段内容，启动一个服务，一般皆为 snlua bootstrap, 即为启动一个 bootstrap 服务。该服务会启动 lua 层相关的基础服务，并且最后启动配置中 start 指定的服务。

start

  1. 创建一个 pthread_t 结构的数组 pid 来保存所有线程的 pid
  2. 创建 monitor *m, 这个变量用来管理全部线程的 monitor
  3. 创建 monitor 线程
  4. 创建 timer 线程
  5. 创建 socket 线程
  6. 创建所有的 worker 线程
  7. 调用 pthread_join 等待全部线程的结束
